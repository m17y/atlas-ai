# Bug 修复记录索引

## 记录规范

每次修复 bug 时，请：
1. 在 `.bug-records/` 目录下创建以日期命名的 Markdown 文件（格式：`YYYY-MM-DD.md`）
2. 文件内容包含：
   - 修复日期
   - 修复的 bug 列表（编号、描述、原因、方案、涉及文件）
   - 新增功能说明
   - 测试链接
   - 下次修复建议
3. **不要修改 bug.md 的内容**，bug.md 只记录修复摘要
4. **不要修改历史记录文件**，保持记录原貌

## 贡献给 AI 的提示

当需要 AI 帮助修复 bug 时：
1. 提供 `bug.md` 文件获取快速修复摘要
2. 提供 `.bug-records/` 下的详细记录了解完整上下文
3. 描述新的 bug 问题，AI 会参考历史记录进行修复
